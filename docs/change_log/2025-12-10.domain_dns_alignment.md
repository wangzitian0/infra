# 2025-12-10: Domain DNS Alignment (i-/x-)

## Summary
Aligned Terraform DNS records and ingress hosts with the internal-domain (no prefix) convention; infra endpoints now live on the internal domain, proxied for 443 services, with `k3s` left DNS-only.

## Changes

### L1 (1.bootstrap)
- `locals.tf`: Domain map uses prefix-less internal hosts; env domain mapping unchanged.
- `2.atlantis.tf`: Atlantis URL/ingress hosts switched to `atlantis.internal`.
- `3.dns_and_cert.tf`: Explicit infra Cloudflare A records with per-host proxy (443 proxied, `k3s` DNS-only); optional internal wildcard only when `INTERNAL_DOMAIN` zone differs; validation checks `atlantis.internal`.
- `variables.tf`: Clarify `internal_domain` / `env_prefix` descriptions.

### L2 (2.platform)
- `2.secret.tf`: Vault ingress/TLS/output hosts updated to `secrets.internal`.
- `3.dashboard.tf`: Dashboard ingress/TLS/output hosts use `kdashboard.internal`.
- `4.kubero.tf`: Kubero UI ingress/TLS/output hosts use `kcloud.internal`.

### Docs
- `1.bootstrap/network.md`, `1.bootstrap/README.md`, `2.platform/README.md`: Document `i-*` infra naming and DNS layout.

## Why
- Keep Terraform implementation consistent with the i-/x- domain architecture documented in `0.check_now.md`.
- Ensure infra stays DNS-only/grey when infra/public share the same Cloudflare zone.
